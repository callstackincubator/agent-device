## 目标

在 H5 页面中，根据关键词（如"面部识别信息"）快速定位到目标位置并截图，替代逐屏滚动方式。

## 当前现状

| 步骤 | 能力 | 状态 |
|------|------|------|
| 获取 H5 全文内容 | webReader 从 `[web]` URL 抓取 | ✅ 已实现 |
| 全文中搜索关键词位置 | webReader 返回 Markdown 文本，可文本匹配 | ✅ 已实现 |
| 计算关键词在页面中的比例位置 | 根据全文段落/字符位置估算百分比 | ✅ 可行 |
| 按比例估算滚动次数 | 总屏数 × 百分比 = 滚动次数 | ✅ 可行 |
| 精确滚动到目标坐标 | 依赖无障碍树暴露 web 内部元素坐标 | ❌ 不可行 |
| 验证关键词是否在可见区域 | `find` 命令搜无障碍树，H5 内部文字不可见 | ❌ 不可行 |

## 核心限制

鸿蒙 H5 页面的 `[web]` 节点不向无障碍树暴露内部文字坐标：

```
# 原生页面
find "设置" → @e3 [button] rect={100, 500, 200, 48}  ✅

# H5 页面
find "面部识别信息" → not found  ❌（web 内容对无障碍树不可见）
```

## 可行方案：按比例滚动 + 人工/视觉确认

```
1. webReader 拿全文 → 文本匹配找关键词位置（如第 5 段/共 13 段 ≈ 38%）
2. 估算总屏数（首次可先全量滚动一次记录）
3. 按比例滚动：13 × 0.38 ≈ 5 次滚动
4. 截图确认（人工或 AI 看图判断关键词是否在画面中）
5. 如未到位，微调 1-2 次滚动
```

效率对比：

| 方式 | 操作次数 | 准确度 |
|------|----------|--------|
| 逐屏滚动 + 截图 | 26+ 条命令 | 100% |
| 按比例滚动 + 微调 | 6-8 条命令 | ~80% |

## 方向2 深入研究：`hdc webinspect` + CDP 注入 JS

### 2026-05-29 调研结果

鸿蒙 ArkWeb 组件支持 Chrome DevTools Protocol（CDP），可通过 `hdc webinspect` 建立调试通道。

### 实现路径

```
路径 A（外部注入，不改 App）：
hdc webinspect                                    # 启用 Web 调试桥
hdc shell cat /proc/net/unix | grep webview       # 找到 WebView PID
hdc fport tcp:9222 localabstract:webview_devtools_remote_<pid>  # 端口转发
→ CDP Runtime.evaluate → 执行 JS 定位文字 → window.scrollTo → 截图

路径 B（App 内 API，需改源码，合规检测不适用）：
WebviewController.runJavaScript('document.title')
```

**合规检测场景只能用路径 A**。

### 关键 JS 片段：定位文字坐标

```javascript
// 在 WebView 中查找目标文字并返回绝对位置
const keyword = "面部识别信息";
const walker = document.createTreeWalker(document.body, NodeFilter.SHOW_TEXT);
let node;
while (node = walker.nextNode()) {
  if (node.textContent.includes(keyword)) {
    const rect = node.parentElement.getBoundingClientRect();
    return JSON.stringify({
      top: rect.top + window.scrollY,  // 绝对 Y 坐标
      left: rect.left + window.scrollX,
      found: true
    });
  }
}
return JSON.stringify({ found: false });
```

### 关键 JS 片段：滚动到目标位置

```javascript
// 滚动到目标文字位置（向上偏移 100px 确保可见）
window.scrollTo({ top: <target_top> - 100, behavior: 'instant' });
```

### 完整工作流设计

```
1. webReader 拿全文 → 识别目标关键词列表
2. snapshot 找 [web] URL → 确认是 H5 页面
3. hdc webinspect → 建立调试通道
4. hdc fport → 端口转发
5. CDP Runtime.evaluate → 注入定位 JS → 返回 { top, left }
6. CDP Runtime.evaluate → window.scrollTo(0, top - 100)
7. agent-device screenshot → 截图作为视觉证据
8. 重复 5-7 直到所有关键词截图完成
```

### 限制

| 限制项 | 说明 |
|--------|------|
| 需要开启 Web 调试 | App 需调用 `webview.WebviewController.enableWebDebugging()`，正式包可能关闭 |
| hdc webinspect 可能不通用 | 部分 HarmonyOS NEXT 版本行为可能不同 |
| 跨域限制 | WebView 内的页面如有 CSP 策略，可能影响 JS 执行 |
| CDP 连接稳定性 | 长时间连接可能断开 |
