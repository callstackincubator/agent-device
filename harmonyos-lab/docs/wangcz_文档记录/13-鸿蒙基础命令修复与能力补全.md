# 鸿蒙基础命令修复与能力补全

| 项 | 内容 |
|----|------|
| 日期 | 2026-05-28 |
| 动机 | `scroll` 方向反了/屏幕尺寸硬编码；`rotate`、`keyboard` 未实现；`doubleTap` 非原生 |
| 生效方式 | `pnpm build` 后 CLI 使用新逻辑 |

---

## 1. 问题诊断

### 1.1 scroll 原始实现问题

```typescript
// src/platforms/harmonyos/input-actions.ts (改前)
const screenCenter = { x: 612, y: 1388 }; // ❌ 硬编码，仅适用 1224x2776

case 'up':
  to = { y: screenCenter.y - distance };  // ❌ 方向反了
case 'down':
  to = { y: screenCenter.y + distance };  // ❌ 方向反了
```

**问题**：
1. 硬编码屏幕中心 `{x:612, y:1388}` — 只适配特定分辨率
2. 方向语义错误 — `scroll up` 手指往上划，实际应手指往下划（内容上滚）
3. 距离固定 500px — 未按屏幕比例计算

### 1.2 其他缺失能力

| 命令 | 原状态 | 问题 |
|------|--------|------|
| `rotate` | ❌ 抛出 "not yet implemented" | 无实现 |
| `keyboard status` | ❌ "only Android/iOS" | 未添加 HarmonyOS 分支 |
| `keyboard dismiss` | ❌ 同上 | 同上 |
| `doubleTap` | ⚠️ 连续两次 tap | 未用 uitest 原生 `doubleClick` |
| `clipboard read` | ❌ | uitest 无 `getClipboard` API |

## 2. 修复方案

### 2.1 scroll 修复

| 文件 | 变更 |
|------|------|
| `src/platforms/harmonyos/input-actions.ts` | +`getHarmonyScreenSize()`；使用 `buildScrollGesturePlan` |

**对比**：

| 项目 | 改前 | 改后 |
|------|------|------|
| 屏幕尺寸 | `{x:612, y:1388}` 硬编码 | `getHarmonyScreenSize()` 动态获取 |
| 方向计算 | 手动算，反了 | `buildScrollGesturePlan`（与 Android/iOS 一致） |
| 参数传递 | 固定 500px | 支持 `amount` / `--pixels` |

**getHarmonyScreenSize 三级回退**：

1. `param get const.display.resolution` → 解析 `WIDTHxHEIGHT`
2. `uitest getWindowInfo` → JSON 解析
3. `snapshotHarmony` → 从根节点 bounds 提取

### 2.2 rotate 实现

```typescript
// src/platforms/harmonyos/input-actions.ts
export async function rotateHarmony(device: DeviceInfo, orientation: DeviceRotation): Promise<void> {
  const rotationValue = mapRotationToValue(orientation); // 0/1/2/3
  
  // 尝试 hidumper
  await runHarmonyHdc(device, ['shell', 'hidumper', '-s', 'DisplayManagerService', '-a', `SetScreenRotation ${rotationValue}`]);
  
  // 回退 param set
  await runHarmonyHdc(device, ['shell', 'param', 'set', 'persist.sys.display.orientation', String(rotationValue)]);
}
```

**注意**：实际旋转需要系统级权限；HDC 命令可能返回成功但屏幕未转（取决于设备策略）。

### 2.3 keyboard 实现

| 文件 | 变更 |
|------|------|
| `src/platforms/harmonyos/input-actions.ts` | +`getHarmonyKeyboardState()` / `dismissHarmonyKeyboard()` |
| `src/core/dispatch.ts` | + HarmonyOS keyboard 分支 |

**检测逻辑**：

```typescript
// 通过 WindowManagerService dump 检测 softKeyboard 窗口
const result = await runHarmonyHdc(device, ['shell', 'hidumper', '-s', 'WindowManagerService', '-a', '-a']);

// 解析窗口列表，找 softKeyboard 窗口的 height
if (line.includes('softKeyboard') && height > 0) → { visible: true, height }
```

**关闭键盘**：调用 `pressBackHarmony()`（Back 键关闭软键盘）。

### 2.4 doubleTap 原生化

```typescript
// 改前
async (x, y) => {
  await pressHarmony(device, x, y);
  await pressHarmony(device, x, y);  // ❌ 两次 tap
}

// 改后
export async function doubleTapHarmony(device: DeviceInfo, x: number, y: number): Promise<void> {
  await runHarmonyHdc(device, ['shell', 'uitest', 'uiInput', 'doubleClick', String(x), String(y)]);
}
```

uitest 支持原生 `doubleClick` 命令，更接近真实双击手势。

## 7. 未实现能力说明

### 7.1 clipboard read

uitest `uiInput` 支持命令：

```
click/doubleClick/longClick <x> <y>
swipe/drag <from_x> <from_y> <to_x> <to_y> [velocity]
keyEvent <keyID/Back/Home/Power>
inputText <x> <y> <text>
text <text>
```

**无 `getClipboard`**。读取剪贴板需要：
- App 级实现（如注入 JS 到 WebView）
- 或调用 `paste` 系统服务（需更高权限）

### 7.2 pinch

uitest 无多点触控 API。`pinch` 在 Android 也未实现（需 instrumentation backend）。
