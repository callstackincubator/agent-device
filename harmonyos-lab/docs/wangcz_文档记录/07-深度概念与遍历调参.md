# 深度概念说明与遍历调参

本文区分两类常被混用的「深度」，说明 `uitest dumpLayout` 能覆盖的范围，以及如何调整智能遍历参数。与 [06-traverse-output报告生成全流程](./06-traverse-output报告生成全流程.md) 配合阅读。

---

## 1. 两种「深度」对照

| 概念 | 英文/代码里叫什么 | 典型数值 | 含义 |
|------|-------------------|----------|------|
| **UI 树深度** | `treeDepth`、`maxDepth`（认知地图统计） | 15～62 层 | 单屏内 ArkUI 组件嵌套：`Stack → Column → … → Text` |
| **遍历深度** | 报告里 `depth: 0/1/2`、`s1_d0` | 默认最多到 `d2` | 从首页**点击进新界面**后，脚本允许再递归几「跳」 |

### 1.1 容易产生的误解

- **误解**：「遍历只有 3 层，所以 `dumpLayout` 也只 dump 了 3 层组件。」  
- **事实**：`dumpLayout` 一次就给出**当前屏完整子树**；`maps/*/cognition-map.json` 里的 `overview.treeDepth` 常为 **15～60+**。  
- **「3」来自哪里**：默认 `TRAVERSE_MAX_DEPTH=2`，允许 `depth` 为 **0、1、2**，报告里常见 **3 个界面**（如 `xhs-smart-v3` 的 `s1_d0` → `s2_d1` → `s3_d2`）。

### 1.2 示意图

```text
【UI 树深度 — 单次 dumpLayout，竖着很深】

  Root
   └─ Stack (depth 2)
        └─ Navigation (depth 3)
             └─ … (depth 15～62)
                  └─ Text "微信登录"

【遍历深度 — 多点几屏，横着走】

  d0 首页 ──点击──► d1 子页 ──点击──► d2 孙页
       │                │
       └── 同层还可点多个入口（广度），见下文 3.2
```

## 2. `uitest dumpLayout` 能做什么、不能做什么

### 2.1 能给的

- **当前时刻、当前前台** 的整棵 ArkUI 树（`children` 递归到底）。
- 仓库里 `scripts/build-cognition-map.js` 会遍历全部子节点，产出：
  - `totalNodes`：节点总数  
  - **`treeDepth`**：单屏 UI 树最深层级  

查看方式：

```bash
node scripts/build-cognition-map.js \
  ./traverse-output/xhs-smart-v3/layouts/initial.json \
  /tmp/cognition-test
# 输出含「树深度: N层」

# 或打开已有
cat ./traverse-output/xhs-smart-v3/maps/s1_d0/cognition-map.json | jq '.overview.treeDepth'
```

### 2.2 不能给的

- **整个 App 所有页面** 合并成一棵树（未打开的 Tab、未进入的设置页不会出现）。
- 需要：**多屏各 dump 一次**，或用智能遍历自动「点进去 → 再 dump」。

| 目标 | 做法 |
|------|------|
| 单屏组件结构、可点元素规划 | 一次 `dumpLayout` + `build-cognition-map` |
| 多屏 App 地图 | 提高遍历参数 + `merge-app-map.js` 合并多份 `cognition-map.json` |

## 3. 遍历深度由谁限制

逻辑在 `scripts/smart-traverse-from-cognition.mjs`：

```javascript
const MAX_DEPTH = Number(process.env.TRAVERSE_MAX_DEPTH || 2);
const MAX_TARGETS_PER_SCREEN = Number(process.env.TRAVERSE_MAX_TARGETS || 12);

// 判定为新界面后，仅当 depth < MAX_DEPTH 才递归
if (isNewScreen && depth < MAX_DEPTH) {
  await exploreScreen(afterLayout, depth + 1, actionDesc);
}
```

`scripts/run-smart-traverse-app.sh` 会把上述环境变量传给该脚本（默认 `2` / `12`）。

### 3.1 `TRAVERSE_MAX_DEPTH` 与屏数关系

| `TRAVERSE_MAX_DEPTH` | 允许的最深 `depth` | 单条点击链上最多界面数 |
|----------------------|--------------------|-------------------------|
| `0` | 仅 `d0` | 1 |
| `2`（默认） | `d0`～`d2` | 3 |
| `5` | `d0`～`d5` | 6 |

报告字段 `screensVisited` **可以大于** 上表：同一 `depth` 上多个入口各走一遍（广度），例如 `dcd-smart-v3` 在 `maxDepth: 2` 下仍访问了 **13** 个界面（`s1_d0` … `s13_d2`）。

### 3.2 新界面判定（影响「走不走深」）

点击后会再次 `dumpLayout`，满足下列条件才视为新界面并可能递归：

- 节点数变化 `> max(15, 原节点数 × 12%)`，**或** 可见 `labels` 指纹变化；  
- 且该指纹未访问过；  
- 且仍在目标 App 前台（`hasTarget`）。

因此：登录墙、协议 H5、点击无反应、仅弹 Toast，都可能让**实际屏数**少于参数上限。

### 3.3 `TRAVERSE_MAX_TARGETS`

每屏从 layout 抽取可点目标后，**最多执行多少次 press**（默认 `12`）。过大时：

- 耗时、误点协议/外链、触发系统弹窗概率上升；  
- 过小则单屏覆盖不全，合并后的 app-map 缺口多。

## 4. 调参命令模板

```bash
pnpm build

export TRAVERSE_DEVICE=<agent-device --device 名>
export TRAVERSE_HDC_TARGET=<hdc -t 设备ID>

# 加深单条链路 + 每屏多试几个入口
export TRAVERSE_MAX_DEPTH=5
export TRAVERSE_MAX_TARGETS=16

./scripts/run-smart-traverse-app.sh \
  com.xingin.xhs_hos \
  EntryAbility \
  ./traverse-output/xhs-deep \
  hs-deep
```

跑完后看：

- `smart-traverse-report.json` → `maxDepth`、`screensVisited`、`results[].depth`  
- `maps/*/cognition-map.json` → `overview.treeDepth`（这是 **UI 树**，不是遍历层）

## 5. 按目标 App 的推荐组合

以下基于本仓库已有 `traverse-output` 实测与护栏行为整理；设备需已解锁、目标 App 已安装。登录/协议未处理时，任何参数都可能卡在首屏。

| App | 包名 | 场景特点 | 推荐 `MAX_DEPTH` | 推荐 `MAX_TARGETS` | 说明 |
|-----|------|----------|------------------|-------------------|------|
| **小红书鸿蒙** | `com.xingin.xhs_hos` | 首屏协议/登录多，外链易误点 | `3`～`4` | `8`～`10` | 默认 `2/12` 曾得 **3 屏**（`xhs-smart-v3`）；略加深便于过协议后进主页，targets 不宜过大以免狂点《用户协议》 |
| **懂车帝** | `com.ss.dcar.auto` | Tab/列表多，同 depth 广度大 | `2`～`3` | `12`～`16` | `dcd-smart-v3` 在 `maxDepth=2` 已 **13 屏**；主要瓶颈是入口数而非 depth，可先 `MAX_TARGETS=14` |
| **国家反诈中心** | `com.hicorenational.antifraud.hmy` | 结构相对扁平 | `2`～`3` | `10`～`12` | `antifraud-smart-v2` 约 **4 屏**；`treeDepth` 约 15，默认深度通常够用 |
| **红果 / 番茄系** | `com.phoenix.read.next` | 信息流、层级多 | `3`～`4` | `14`～`18` | 参考 `hongguo-smart`（`maxDepth=3` → **12 屏**）；内容 App 宜略增 targets |
| **滴滴出行** | `com.sdu.didi.hmos.psnger` | 首启链长：隐私→位置→系统通知；16+ 业务格 + 5 Tab | `4`～`8` | `16`～`20` | **脚本改完前**先用 CLI 走通首启（[11](./11-实操案例-滴滴首启与CLI探索.md)）；默认脚本 `press` 易 1 击即停（[12](./12-遍历脚本已知问题与改法.md)） |
| **仅要单屏结构图** | 任意 | 不自动点按 | — | — | 手动 `hdc uitest dumpLayout` 或 `snapshot --json`，再 `build-cognition-map.js` |

### 5.1 一键示例（复制即用）

```bash
# 小红书：略加深、少点协议链
TRAVERSE_MAX_DEPTH=4 TRAVERSE_MAX_TARGETS=9 \
  ./scripts/run-smart-traverse-app.sh com.xingin.xhs_hos EntryAbility ./traverse-output/xhs-deep hs-xhs

# 懂车帝：保持 depth，加大每屏入口
TRAVERSE_MAX_DEPTH=2 TRAVERSE_MAX_TARGETS=14 \
  ./scripts/run-smart-traverse-app.sh com.ss.dcar.auto EntryAbility ./traverse-output/dcd-wide hs-dcd

# 反诈：默认附近即可
TRAVERSE_MAX_DEPTH=2 TRAVERSE_MAX_TARGETS=12 \
  ./scripts/run-smart-traverse-app.sh com.hicorenational.antifraud.hmy EntryAbility ./traverse-output/antifraud hs-af

# 红果：加深 + 多目标
TRAVERSE_MAX_DEPTH=4 TRAVERSE_MAX_TARGETS=16 \
  ./scripts/run-smart-traverse-app.sh com.phoenix.read.next EntryAbility ./traverse-output/hongguo-deep hs-hg

# 滴滴：脚本改完后（launchAbility 可省略）
TRAVERSE_MAX_DEPTH=8 TRAVERSE_MAX_TARGETS=16 \
  ./scripts/run-smart-traverse-app.sh com.sdu.didi.hmos.psnger ./traverse-output/didi-smart hs-didi
# 改脚本前请用 CLI 探索，见 11-实操案例-滴滴首启与CLI探索.md
```

### 5.2 调参顺序建议

1. 先默认跑一轮，看 `smart-traverse-report.md` 里 **新界面 / 无变化 / 跳过** 比例。  
2. 若大量 **无变化**：检查登录墙、弹窗（见 [05](./05-实操案例-登录与弹窗.md)），或略增 `MAX_TARGETS`。  
3. 若 **单链只走到 d2 就停**、但仍有明显未进子页：增大 `MAX_DEPTH`。  
4. 若 **误点协议/跳出 App**：减小 `MAX_TARGETS`，或先手动 `press` 过协议再遍历。  
5. 合并多屏：`node scripts/merge-app-map.js maps/s1_d0/cognition-map.json maps/s2_d1/... ./traverse-output/app-map.json`。

## 6. 与 `snapshot -i` 的关系（复习）

| | `dumpLayout` / 遍历 layouts | `snapshot -i` |
|--|----------------------------|---------------|
| 树是否完整 | 当前屏 **完整** JSON | **过滤后**可操作列表 |
| 深度指标 | `treeDepth` 十几～几十 | 行数少，不代表树浅 |
| 典型用途 | 认知地图、坐标规划 | Agent 点按、`press @eN` |

详见 [02-snapshot-i与鸿蒙快照](./02-snapshot-i与鸿蒙快照.md)。

## 7. 速查结论

| 问题 | 答案 |
|------|------|
| `dumpLayout` 能否拿到单屏 UI 最深层级？ | **能**，看 `cognition-map` 的 `treeDepth` |
| 为何报告里只有 3 个界面？ | 默认 **`TRAVERSE_MAX_DEPTH=2`**，单链最多 3 跳；与 UI 树深度无关 |
| 能否一次 dump 整个 App？ | **不能**，需多屏遍历 + 合并 |
| 想逛更多页面？ | 提高 `TRAVERSE_MAX_DEPTH` 和/或 `TRAVERSE_MAX_TARGETS`，并处理登录/协议 |
