## 2. 逻辑对比（`shouldIncludeNode`）

### 改前

- `hittable` 或 `scrollable` → 保留  
- **任意** 非空 `label` 或 `identifier` → 保留（噪音大）

### 改后

- `hittable` 或 `scrollable` → 保留  
- `label` 非空且 `ancestorInteractive`（祖先有可点/可滚）→ 保留  
- `identifier` 非空且 `descendantInteractive`（后代有可点/可滚）→ 保留  
- 其余 → 丢弃  

遍历 walk 时传递 `ancestorInteractive`，在子树内传递 `hittable || scrollable`。

---

## 3. 弹窗 scope（`findBestFocusedInteractiveRoot`）

当 `interactiveOnly && !raw` 时：

1. 在全树找 **`focused === 'true'`** 且子树含可交互节点的节点  
2. 取 **depth 最深** 的一个作为 scope 根（多为 `navdestination` 弹窗层）  
3. **只遍历该子树**；scope 根入口设 `ancestorInteractive = true`，保留弹窗标题等文案  

**预期效果**：隐私弹窗时 `-i` 从 ~49 降到 ~13，且不再列出背景「其他登录方式」「微信登录」等。

## 5. 使用注意

1. **必须** `pnpm build`，否则仍跑旧 `dist`  
2. `[text]` + `[button]` 并存时，优先 button 或 `label="…"`  
3. 若 scope 未触发（无 focused 标记），行为与「仅收紧过滤」一致，节点仍可能比 Android 多  
4. 疑难用 `snapshot -i --json` 核对 `focused`、`hittable`、`rect`
