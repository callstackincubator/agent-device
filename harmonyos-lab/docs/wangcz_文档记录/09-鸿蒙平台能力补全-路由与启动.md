## 1. 平台路由修复（致命）

### 1.1 问题

`apps`、`appstate` 在 HarmonyOS session 下曾错误调用 Android 实现（`listAndroidApps` / `getAndroidAppState`），表现为 hdc 设备上仍走 adb 语义或返回空/错包。

### 1.2 修复

| 命令 | Handler | 鸿蒙实现 |
|------|---------|----------|
| `apps` | `session-inventory.ts` | `listHarmonyApps` |
| `appstate` | `session-state.ts` | `getHarmonyAppState` |

单测：

- `src/daemon/handlers/__tests__/session-inventory-harmonyos.test.ts`
- `src/daemon/handlers/__tests__/session-state-harmonyos.test.ts`

**规则**：凡 HarmonyOS 命令，能力判断与 dispatch 以 `src/core/capabilities.ts` 为准，handler 内勿再硬编码 Android 分支。

## 2. wukong appinfo 与 launchAbility

### 2.1 背景

鸿蒙 `open <bundle>` 常需 Ability 名（如 `EntryAbility`）。设备上可通过：

```bash
hdc shell wukong appinfo
```

解析 Bundle → 默认启动 Ability 映射。

### 2.2 代码

| 文件 | 作用 |
|------|------|
| `src/platforms/harmonyos/launch-abilities.ts` | 缓存 5 分钟；`getHarmonyLaunchAbilities` / `lookupHarmonyLaunchAbility` |
| `src/platforms/harmonyos/app-parsers.ts` | 解析 wukong 输出 |
| `src/platforms/harmonyos/app-lifecycle.ts` | `openHarmonyApp` 策略链 |

**open 回退顺序**（简化）：

1. 用户指定 `--activity`
2. wukong 缓存中的 launchAbility
3. `MainAbility` / `EntryAbility` 试探
4. 仅包名启动

### 2.3 CLI：`apps --json`

`apps --json` 现返回 `appDetails`，含 `launchAbility` 字段；`listHarmonyApps` 合并到 `activity` 字段。

```bash
node dist/src/cli.js apps --platform harmonyos --device <hdc-serial> --json
```

遍历脚本辅助（非 CLI 本体）：

- `scripts/resolve-harmony-launch-ability.mjs` — shell 解析 launchAbility
- `run-smart-traverse-app.sh` — Ability 参数可省略，自动从 `apps --json` 解析

### 2.4 实测包名（设备 `22M0223824043030`）

| App | 包名 | launchAbility |
|-----|------|---------------|
| 滴滴出行 | `com.sdu.didi.hmos.psnger` | `EntryAbility` |
| 懂车帝 | （示例） | `DcarAbility` |

## 3. App 存储清理（重置首启）

### 3.1 设备命令

```bash
hdc shell bm clean -n <包名> -d   # data
hdc shell bm clean -n <包名> -c   # cache
```

隐私同意、部分权限弹窗会写入 storage；**不清 data 则无法复现首启弹窗**。

### 3.2 代码

| 文件 | 作用 |
|------|------|
| `src/platforms/harmonyos/app-lifecycle.ts` | `clearHarmonyAppStorage()`：force-stop + bm clean |
| `scripts/clear-harmony-app-storage.mjs` | 遍历前调用 |
| `src/platforms/harmonyos/__tests__/app-storage.test.ts` | 单测 |

**注意**：agent-device **尚无**独立 `clear-app-data` CLI 子命令；手操可用 hdc，批量遍历由 `run-smart-traverse-app.sh` 默认 `TRAVERSE_CLEAR_APP_STORAGE=1` 触发。

跳过清理：

```bash
TRAVERSE_CLEAR_APP_STORAGE=0 ./scripts/run-smart-traverse-app.sh ...
```

## 4. uitest 预检

### 4.1 问题

设备上若残留 `uitest uiRecord record`（早期 probe 未退出），`dumpLayout` / `snapshot` 会超时或产出 0 字节 JSON。

### 4.2 代码

| 文件 | 作用 |
|------|------|
| `src/platforms/harmonyos/uitest-preflight.ts` | 检测 stuck uiRecord；`ensureHarmonyUitestReady` |
| `src/platforms/harmonyos/snapshot.ts` | snapshot 前预检 |
| `scripts/ensure-harmony-uitest-ready.mjs` | 遍历脚本入口预检 |

**处理**：重启设备清 stuck 进程；勿长期后台跑 `uitest uiRecord record`。
