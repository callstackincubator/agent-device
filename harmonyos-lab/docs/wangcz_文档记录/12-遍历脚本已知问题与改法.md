## 1. 现象：循环不起来

### 1.1 脚本侧日志（典型）

```text
=== 界面 s1_d0 (depth=0) nodes=66 ===
  弹窗模式: 仅点击同意/确定类按钮
  计划点击: 1 个
  -> 点击 1/1: [modal-accept] press 540 1722 (同意并开始使用)
     ! 点击后离开目标应用 (wrong_app)，不递归
  [recover] wrong_app → 重新拉起
=== 完成 ===
界面: 1, 点击: 1, 新界面: 0, 跳过: 1
```

### 1.2 CLI 侧（同屏同按钮）

```bash
click @e11   # 「同意并开始使用」
appstate     # 仍为 com.sdu.didi.hmos.psnger → 继续位置权限 → 主界面
```

**根因**：不是滴滴无法遍历，而是脚本 **交互方式 + modal 单点退出** 与 CLI 不一致。

## 2. 问题清单与改法

| # | 现象 | 脚本现状 | 建议改法 |
|---|------|----------|----------|
| 1 | 点「同意」进 `privacycenter` | `press cx cy` 裸坐标 | 走 daemon **`click @ref` / `click label=`**（与 CLI 同路径） |
| 2 | 同意 1 次就结束 | `hasModal` 时 `plan.length===1`，wrong_app 后 `break` | 同意后 **re-dump + 继续本屏** 或递归 `exploreScreen` |
| 3 | 系统通知弹窗漏检 | 规划基于 dumpLayout，但 agent 读 `-i` 会漏 | 权限链阶段检测 **uiextension/dialog** 全树节点 |
| 4 | 权限未串行 | 只认 App 内隐私文案 | 固定阶段：**隐私 → 位置(应用内) → 通知(系统)** |
| 5 | `back` 回不到 Tab | 子页 back 后 fingerprint 变但无 Tab | recover 用 **`open` / force-stop + open** |
| 6 | `wrong_app` 误杀 | layout bundles 含系统包即 recover+break | 区分 **短暂 overlay** vs **真正切 App**；`click` 路径 rarely wrong_app |
| 7 | Tab 点不到 | `label=车主` 在子页失败 | 先 recover 到主界面再点 Tab；或缓存 Tab 坐标 |
| 8 | 每轮清 data | 默认 `bm clean` | **仅首轮**清；后续 force-stop 保留已授权 |
| 9 | 深度不够 | 默认 `MAX_DEPTH=2` | 滴滴类：**depth≥4**，`MAX_TARGETS≥16` |

## 3. 脚本 vs CLI 职责划分

| | 智能遍历脚本 | CLI 手操 |
|--|--------------|----------|
| UI 来源 | `dumpLayout` 完整 JSON | `snapshot -i` / `snapshot` |
| 点击 | 现为 `press x y` | `click @ref` / `click label=` |
| 规划 | `build-cognition-map` + 规则 | 人/Agent 读 snapshot 决策 |
| 适用 | 批量报告、留痕目录 | 验证交互、摸清首启链、反推脚本改法 |

**原则**：脚本应复用 **与 CLI 相同的 daemon 交互路径**（click/fill），而不是绕过 selector 直接 uiInput 坐标。
