# 实操案例：登录与弹窗

典型路径：**登录页 → 点「微信登录」→ 隐私协议弹窗 → 点「同意并继续」**。

---

## 1. 登录页：点「微信登录」

```bash
node bin/agent-device.mjs open <包名> --platform harmonyos --session hs1

node bin/agent-device.mjs snapshot -i --platform harmonyos --session hs1

# 推荐：不依赖 @e 编号
node bin/agent-device.mjs press 'label="微信登录"' --platform harmonyos --session hs1

# 备选：点 button 父节点（勿只点 [text]）
node bin/agent-device.mjs press @e10 --platform harmonyos --session hs1

node bin/agent-device.mjs snapshot -i --platform harmonyos --session hs1
```

---

## 2. 隐私弹窗出现后

### 界面特征

- 文案：`请阅读并同意以下条款`、`不同意`、`同意并继续`  
- 结构：新的 `navdestination` + `[focused]`（见附录实录）

### 应执行

```bash
node bin/agent-device.mjs press 'label="同意并继续"' --platform harmonyos --session hs1
# 或 press @e13（编号以当前 -i 为准，build 后弹窗约 13 nodes）

node bin/agent-device.mjs snapshot -i --platform harmonyos --session hs1
```

### 不要做的事

| 错误操作 | 现象 |
|----------|------|
| `press @e15`（背景「其他登录方式」） | 可能 `Tapped` 成功，**界面不变** |
| 不重新 `snapshot -i` 就用旧 @eN | ref 已过期或指向错误层 |
| 未 `pnpm build` 却以为已 scope | 仍可能看到 ~49 nodes 含背景 |

---

## 3. 现象对照表

| 现象 | 原因 | 处理 |
|------|------|------|
| `Tapped` 但无变化 | 点在模态层下方的背景 | 只操作弹窗内控件 |
| `-i` 仍有背景项 | 旧 dist / 无 focused scope | `pnpm build` 后重试 |
| 点 `@e13`「微信登录」后出现弹窗 | 正常 | 下一步点「同意并继续」 |
| `-i` 约 13 nodes | scope 生效 | 按列表操作即可 |

---

## 4. 排查 checklist

- [ ] 已 `pnpm build`  
- [ ] 已 `open` 且 `--session` 一致  
- [ ] 每次 UI 变化后重新 `snapshot -i`  
- [ ] 优先 `label="..."`，其次 `@eN`，再试父级 `[button]`  
- [ ] 用 `is visible 'label="同意并继续"'` 或 `wait` 验证  
- [ ] 疑惑时：`snapshot -i --json`
